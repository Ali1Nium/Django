جنگو (Django)
هدف: جنگو یک فریمورک وب سطح بالا برای ساخت برنامه‌های وب است. این فریمورک طراحی شده است تا به توسعه‌دهندگان کمک کند وب‌سایت‌های قوی، مقیاس‌پذیر و قابل نگهداری را به سرعت ایجاد کنند.
ویژگی‌ها:
معماری MVC: جنگو از الگوی Model-View-Controller (MVC) پیروی می‌کند (که در اصطلاح جنگو به Model-View-Template (MVT) معروف است).
ORM: جنگو دارای سیستم Object-Relational Mapping (ORM) برای تعامل با پایگاه‌های داده با استفاده از کد پایتون به جای SQL است.
رابط مدیریت: یک رابط مدیریت به‌طور خودکار برای مدیریت داده‌های برنامه فراهم می‌کند.
سیستم قالب‌ها: یک موتور قالب‌سازی داخلی برای رندر کردن HTML دارد.
مسیر‌یابی: مسیردهی URL برای نقشه‌برداری URL‌ها به ویوها استفاده می‌شود.
Django REST Framework (DRF)
هدف: DRF یک جعبه‌ابزار قدرتمند و انعطاف‌پذیر برای ساخت وب API‌ها در جنگو است. این فریمورک جنگو را برای ایجاد API‌های RESTful گسترش می‌دهد.
ویژگی‌ها:
سریال‌سازی: DRF شامل سریالایزرهایی است که داده‌های پیچیده (مانند مدل‌های جنگو) را به JSON، XML یا فرمت‌های دیگر و بالعکس تبدیل می‌کنند.
ViewSets: DRF ViewSets را ارائه می‌دهد که عملیات CRUD را مدیریت کرده و ایجاد نقاط پایان API را آسان می‌کند.
احراز هویت و مجوزها: شامل پشتیبانی داخلی برای طرح‌های احراز هویت مختلف (مانند احراز هویت توکن، OAuth) و مجوزها است.
API قابل مرور: یک رابط وب‌پذیر برای نقاط پایان API ارائه می‌دهد که تعامل با API را برای توسعه‌دهندگان آسان‌تر می‌کند.
صفحه‌بندی و فیلتر کردن: از صفحه‌بندی و فیلتر کردن نتایج پرس‌وجو پشتیبانی می‌کند.
تفاوت‌های کلیدی
دامنه:

جنگو: فریمورک چندمنظوره برای ساخت برنامه‌های وب کامل، از جمله مدیریت منطق سمت سرور، تعامل با پایگاه داده و رندر کردن HTML.
DRF: افزونه‌ای تخصصی برای جنگو که بر روی ایجاد API‌های RESTful تمرکز دارد.
استفاده:

جنگو: برای توسعه وب‌سایت‌های سنتی با حساب‌های کاربری، پنل مدیریت و صفحات تعاملی استفاده می‌شود.
DRF: برای نمایش داده‌ها و قابلیت‌های برنامه جنگو از طریق API‌ها استفاده می‌شود، معمولاً برای برنامه‌های سمت مشتری (مانند برنامه‌های تک‌صفحه‌ای) یا برای ادغام با خدمات دیگر.
اجزای اصلی:

جنگو: شامل اجزای مانند مدل‌ها، ویوها، قالب‌ها و مسیر‌یابی است.
DRF: اجزای خاص API مانند سریالایزرها، viewsets، routers و مکانیزم‌های احراز هویت API را اضافه می‌کند.
خروجی:

جنگو: به‌طور کلی HTML را خروجی می‌دهد و صفحات وب را مدیریت می‌کند.
DRF: داده‌ها را در فرمت‌های JSON، XML یا دیگر فرمت‌های مناسب برای مصرف API خروجی می‌دهد.
مثال‌های استفاده
جنگو: ساخت وب‌سایت سنتی با حساب‌های کاربری، پنل مدیریت و صفحات تعاملی.
DRF: ایجاد یک API RESTful که به یک برنامه موبایل یا یک فریمورک فرانت‌اند (مانند React یا Vue.js) اجازه می‌دهد با برنامه جنگو تعامل کند.
به طور خلاصه، جنگو پایه‌ای برای ساخت برنامه‌های وب فراهم می‌کند، در حالی که DRF قابلیت‌های جنگو را برای پشتیبانی از ایجاد API‌های RESTful گسترش می‌دهد. این دو با هم به خوبی مکمل یکدیگر هستند وقتی که به هر دو برنامه وب و API RESTful نیاز دارید.
